Video transformation is a resource-intensive operation, so it takes a significant amount of time and heavily depends on the size of the original file. It is recommended to use <a href="../async-transform/">asynchronous transformation</a> for video processing.

If the video file has not yet been transformed, a response with a `423` status code will be sent, indicating that the file is not yet ready.

Example:
```
// optimize video size ?q_a
https://api.defaultuploader.com/v1/video/upload/token/sample.mp4?q_a
```

### q_auto
<Badge text="alias: q_a" variant="caution" size="small" />

Sets optimal video settings, preserving quality while reducing file size.
```
?q_a
```

<Video sources={exampleSource.videos} param="q_a" />

<Aside title="The video size was reduced by 7 times!" type="tip">
  The original video size was 50MB, and after optimization, it became **7MB**.
</Aside>

### format
<Badge text="alias: f" variant="caution" size="small" />

Changes the video format.
```
Supported formats
MP4, MOV, WEBM, AVI, default is mp4
```

### vertical
<Badge text="alias: vt" variant="caution" size="small" />

Creates a video in a 9x16 format by cropping the center.
```
?vt
```

<Video sources={exampleSource.videos} param="vt" />

<Aside title="Perfect for Social Media" type="tip">
  Powerful video optimization from 50MB to **2.94MB** and the popular 9x16 format for mobile devices.
</Aside>

### thumbnail

Creates a thumbnail image from the video.

```
?thumbnail
By default, a frame from the first 100ms of the video is used.
```

<Image sources={exampleSource.videos} param="thumbnail" />

### duration

Specifies the moment in the video to create the thumbnail.

```
1 = 100ms
Works only with the thumbnail property.
```

<Image sources={exampleSource.videos} param="thumbnail&duration=311" />
